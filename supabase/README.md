# Supabase 시작 가이드

이 폴더는 `잉크로드` MVP용 백엔드 시작점입니다.

## 무엇이 들어 있나요?

- `schema.sql`
  - 테이블 구조
  - 인덱스
  - 기본 RLS 정책
  - 회원가입 시 프로필 자동 생성 트리거
- `seed.sql`
  - 테스트용 작가
  - 테스트용 작품
  - 테스트용 회차
  - 테스트용 태그와 이벤트전

## 가장 쉬운 시작 순서

1. Supabase 프로젝트를 만듭니다.
2. Supabase SQL Editor를 엽니다.
3. `schema.sql` 내용을 먼저 실행합니다.
4. 테스트 데이터가 필요하면 `seed.sql`을 실행합니다.
5. 프론트에서 실제로 붙이려면 `assets/supabase-config.js`에 publishable key를 넣습니다. URL과 project ref는 이미 채워뒀습니다.

## 이 스키마로 바로 할 수 있는 것

- 회원가입 / 로그인
- 작품 목록 불러오기
- 작품 상세 불러오기
- 회차 목록 불러오기
- 무료 회차 본문 불러오기
- 태그 검색
- 이벤트전 목록
- 서재 저장
- 읽던 위치 저장
- 찜하기

## 주의할 점

- `purchases` 테이블은 나중에 결제 완료 후 서버 함수로만 넣는 것이 안전합니다.
- 유료 회차 본문은 `episode_contents`에 따로 분리되어 있어서, 권한 없이 바로 읽히지 않게 해두었습니다.

- 프론트는 `assets/supabase-config.js`가 비어 있으면 기존 정적 화면으로 남고, 값을 넣으면 Supabase 데이터를 읽기 시작합니다.
