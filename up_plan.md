# InkRoad 현재 작업 계획

## 지금 왜 다시 정리하나

최근 구현이 많이 늘어나면서, 원래 원했던 방향과 실제 화면이 조금 달라졌습니다.

특히:
- `작품 업로드 화면`은 로그인 박스가 화면 안에 들어와서 어색해졌고
- `회차 업로드 화면`은 글쓰기 화면보다 관리 콘솔 느낌이 강해졌습니다

그래서 지금부터는 `원래 설계안에 맞게 다시 정리하는 것`을 우선합니다.

---

## 현재 상태 요약

### 이번에 반영한 것

- `회차 업로드 화면`
  - 에디터 중심 구조로 다시 정리함
  - 로그인은 업로드 화면 안에서 처리하지 않고 `auth_pc.html`로 분리함
  - 관리 기능은 보조 영역으로 내림

- `작품 업로드 화면`
  - 제목 / 소개 / 첫 회차를 메인으로 올리고
  - 표지 / 설정 / 태그는 보조 영역으로 내림

- `뷰어`
  - Markdown 렌더링을 뷰어 전용 함수로 정리함
  - 제목 / 굵게 / 목록 / 인용 / 코드 / 장면 전환 스타일 추가
  - 복사 방지 이벤트 중복 바인딩도 막음

### 이미 되어 있는 것

- 홈 / 탐색 / 상세 / 뷰어 / 로그인 / 서재 / 작가 대시보드 화면
- 작품 업로드 / 회차 업로드 기본 기능
- Supabase 연결
- 회차 스냅샷 테이블
- 에디터 자동 저장 관련 로직 일부
- 버전 비교(diff) / 복원 기능 일부

### 지금 문제인 것

- `작품 업로드 화면`
  - 로그인 UI가 한 화면 안에 보이던 흐름이 어색했음
  - 마크다운 체감이 약했음
  - 태그 수가 부족했음

- `회차 업로드 화면`
  - 원래는 `에디터 중심`이어야 했는데
  - 지금은 `회차 관리 콘솔` 성격이 너무 강함
  - 즉, 글쓰기보다 관리 기능이 더 눈에 들어옴

---

## 가장 중요한 기준

앞으로는 아래 기준을 우선합니다.

1. 업로드 화면은 `관리 화면`보다 `작성 화면`이어야 함
2. 로그인은 `별도 인증 화면`에서 처리하고, 업로드 안에서는 보여주지 않음
3. 작품 업로드는 `제목 / 소개 / 태그 / 첫 회차 작성`에 집중
4. 회차 업로드는 `에디터 / 자동 저장 / 버전 복원`이 중심
5. 숨김 / 휴지통 / 일괄 작업 같은 운영 기능은 보조이거나 별도 관리 화면으로 분리

## 바로 다음 작업 우선순위

### 1순위: 회차 업로드 화면 원래 설계안으로 재정렬

상태:
- 완료

목표:
- `글쓰기 중심 화면`으로 다시 정리

핵심 작업:
- 오른쪽 관리 패널의 비중 축소
- 회차 목록 / 숨김 / 휴지통 / 일괄 관리 기능 정리
- 스냅샷 패널과 diff 비교를 에디터 보조 기능으로 유지
- 발행 버튼 / 저장 상태 / 자동 저장 안내를 더 분명하게 표시

유지할 것:
- Toast UI Editor
- 위지윅 / 마크다운 토글
- 자동 저장
- localStorage 폴백
- snapshot / diff / 복원

줄일 것:
- 관리 콘솔 느낌
- 회차 운영 기능 과노출

### 2순위: 작품 업로드 화면 마감

상태:
- 1차 완료

목표:
- `정말 작품 쓰는 첫 화면`처럼 보이게 정리

핵심 작업:
- 로그인은 업로드 페이지 내부에 절대 넣지 않기
- 마크다운 미리보기 체감 더 강화
- 태그 선택 UX 정리
- 소개 / 첫 회차 입력 영역 가독성 정리

### 3순위: 뷰어 쪽 마크다운 렌더링 확인

상태:
- 1차 완료

목표:
- 작가가 쓴 문법이 독자 화면에서 자연스럽게 보이게 하기

확인할 것:
- 제목
- 굵게
- 기울임
- 목록
- 인용
- 장면 전환

### 4순위: 운영 기능은 분리해서 다시 배치

상태:
- 다음 작업

대상:
- 숨김
- 휴지통
- 전체 복원
- 비우기
- 보관 / 보관 해제

원칙:
- 작성 화면에서 너무 많이 보이지 않게 하기
- 필요하면 `작품 관리` 쪽으로 이동

## 지금 파일별 정리 방향

### `novel_upload_pc.html`

- 작품 등록 전용 화면으로 유지
- 로그인 UI 제거 유지
- 소개 / 태그 / 첫 회차 작성 흐름 강화

### `episode_upload_pc.html`

- 가장 크게 다시 정리할 대상
- 에디터 중심 구조로 재배치
- 회차 관리 기능은 축소 또는 분리

### `assets/upload-studio.js`

- 작품 업로드 전용 로직 유지
- 로그인 리다이렉트 / 마크다운 미리보기 / 태그 저장 안정화

### `assets/episode-upload.js`

- 자동 저장 / snapshot / diff / restore는 유지
- 관리 기능은 구조 다시 정리

### `supabase`

- 이미 연결됨
- 스냅샷 / 태그 저장 관련 반영 상태 확인 완료
- 앞으로는 화면 정리 후 필요한 DB 수정만 최소로 추가

## 작업 순서

1. `episode_upload_pc.html`을 원래 설계안 기준으로 다시 설계
2. 그 설계에 맞춰 `assets/episode-upload.js` 정리
3. 화면 확인
4. 그다음 `novel_upload_pc.html` 마감 조정
5. 마지막으로 뷰어 렌더링 점검
