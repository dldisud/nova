# INKROAD i18n (Internationalization) Design Spec

**Date:** 2026-04-04
**Scope:** Add multi-language support (Korean + English) to all 16 pages
**Stack constraint:** Pure HTML/CSS/JS, no build tools, CDN only

---

## 1. Overview

INKROAD is a global web novel marketplace. All UI text is currently hardcoded in Korean. This spec adds a lightweight i18n system that:

- Auto-detects browser language on first visit
- Allows manual language switching via header dropdown
- Translates ~60 unique UI text strings across 16 HTML pages and 7 JS files
- Does NOT translate user content (novel titles, bodies, author names)

---

## 2. Architecture

### 2.1 Core Module: `assets/i18n.js`

A self-contained IIFE (~120 lines) exposing `window.inkroadI18n`:

```
window.inkroadI18n = {
  locale: "ko",
  dict: {},
  init(),
  t(key, params),
  applyDOM(),
  setLocale(lang),
  getLocale()
}
```

**Language detection priority:**
1. `localStorage.getItem("inkroad-lang")` — user's previous manual choice
2. `navigator.language` first 2 chars — browser language
3. Fallback `"en"` if detected language is not in supported list

**Supported languages (initial):** `["ko", "en"]`

### 2.2 `t(key, params)` Function

- Looks up `key` in `dict` using dot notation: `t("editor.save")` finds `dict.editor.save`
- Returns the value string, or the key itself if not found (+ `console.warn`)
- Simple interpolation: `t("common.view_count", { n: 42 })` replaces `{{n}}` with `42`
- Template syntax: `{{variable}}` only

### 2.3 `applyDOM()` Function

Scans the document for translation attributes:

| Attribute | Target property |
|-----------|----------------|
| `data-i18n="key"` | `element.textContent` |
| `data-i18n-placeholder="key"` | `element.placeholder` |
| `data-i18n-title="key"` | `element.title` |
| `data-i18n-html="key"` | `element.innerHTML` (for strings with simple markup) |

### 2.4 `setLocale(lang)` Function

1. Set `locale = lang`
2. Save to `localStorage.setItem("inkroad-lang", lang)`
3. `await fetch("assets/i18n/" + lang + ".json")` to load new dictionary
4. Call `applyDOM()` to re-scan static HTML elements
5. Dispatch `window.dispatchEvent(new Event("inkroad-locale-change"))` for JS-rendered pages to re-render

## 3. Dictionary Files

### 3.1 File Structure

```
assets/
  i18n/
    ko.json
    en.json
```

### 3.2 Key Groups (6 groups, ~60 keys)

**`common`** — Shared across multiple pages:
- confirm, cancel, save, delete, loading
- won (currency suffix), episode_unit ("화"/"Ep."), per_episode, free, paid
- view_count ("조회 {{n}}" / "{{n}} views")
- unknown_author ("작가 미상" / "Unknown author")

**`nav`** — Navigation and header:
- home, search, my_works, my_library, upload
- login, logout, language

**`auth`** — Authentication forms:
- login_title, email, password, nickname
- signup, login_button, logged_in
- login_required, signup_submitted
- config_missing (Supabase connection message)

**`editor`** — Episode upload/editing:
- wysiwyg, text_mode, history, version_history
- unsaved, saved, saving, offline_saved, draft_restored
- publish, publishing, moving
- restore_confirm, no_snapshots, snapshot_label_prompt, snapshot_published
- select_work, episode_title, episode_body, access_free, access_paid
- episode_price, publish_success, publish_error

**`viewer`** — Novel reader:
- free_episode, paid_episode, prev, next, toc, first_episode
- bookmark_add, bookmark_done, buy
- paywall_title, paywall_message
- body_loading, body_unavailable
- filter_instruction, filter_include, filter_exclude, filter_empty

**`status`** — Novel/work status labels:
- serializing, completed, draft, hiatus

**`genre`** — Genre filter labels:
- fantasy, romance, regression, academy, translated, dark, family, dark_fantasy

### 3.3 Naming Rules

- 2-level depth only: `group.key`
- Snake_case for keys
- No user content in dictionaries (novel titles, bodies, author names)
- Genre labels included because they are UI filter labels, not user content

## 4. Language Picker UI

### 4.1 PC (topbar)

Position: Inside `.nav-actions`, before the first action button.

Markup: Empty container `<div data-lang-picker></div>` in HTML. `i18n.js` injects:

```html
<button class="lang-picker-toggle">
  <span class="material-symbols-outlined">language</span>
  <span data-lang-current>KO</span>
</button>
<div class="lang-picker-dropdown" hidden>
  <button data-lang-option="ko">한국어</button>
  <button data-lang-option="en">English</button>
</div>
```

Behavior: Click toggle → show/hide dropdown. Click option → `setLocale(lang)`. Click outside → close.

### 4.2 Mobile (header)

Position: Inside mobile header, right-aligned icon area.

Same `<div data-lang-picker></div>` container. i18n.js detects mobile via `document.body.classList.contains("mobile-app")` and renders a compact toggle:

```html
<button class="lang-picker-compact" data-lang-compact>KO</button>
```

Click cycles through supported languages: KO → EN → KO.

### 4.3 Styling

Added to `assets/styles.css`. Follows existing design system:
- PC: Ghost button style matching `.nav-actions` buttons
- Mobile: Small text button, same color as header icons
- Dropdown: Same surface/border as existing dropdowns in the project
- No purple/neon/gradient (per project design rules)

## 5. Integration with Existing JS Files

### 5.1 Script Load Order (all pages)

```
i18n.js              ← NEW, first to load
supabase-config.js
app.js
supabase-js (CDN)
supabase-live.js
[page-specific JS]
```

`i18n.js` runs `init()` at load, which is async. Other scripts access `t()` synchronously because by the time they execute, `init()` has already resolved (scripts load sequentially).

### 5.2 Changes Per JS File

**`store-redesign.js`** (~30 string replacements):
- All hardcoded Korean strings → `t()` calls
- `formatCount()` locale → `inkroadI18n.locale`
- `wonFormat()` currency suffix → `t("common.won")`
- Status labels → `t("status." + key)`
- Genre labels → `t("genre." + key)`
- Add `window.addEventListener("inkroad-locale-change", ...)` to re-render current page
- `Intl.NumberFormat("ko-KR")` → `Intl.NumberFormat(inkroadI18n.locale)`

**`episode-upload.js`** (~25 string replacements):
- Status labels, button text, placeholders, messages → `t()` calls
- `Intl.NumberFormat("ko-KR")` → `Intl.NumberFormat(inkroadI18n.locale)`
- AM/PM labels → `t()` or use `Intl.DateTimeFormat`

**`app.js`** (~15 string replacements):
- Bookmark labels, filter instructions, currency formatting → `t()` calls

**`creator-dashboard.js`** and **`upload-studio.js`**:
- Same pattern: find Korean strings, replace with `t()` calls

**`supabase-live.js`** and **`supabase-config.js`**:
- No UI text, no changes needed

### 5.3 Changes Per HTML File (16 files)

For each HTML file:
1. Add `<script src="assets/i18n.js"></script>` as first script (before supabase-config.js)
2. Add `<div data-lang-picker></div>` in header/nav area
3. Add `data-i18n="key"` attributes to static Korean text elements
4. Add `data-i18n-placeholder="key"` to input/textarea elements with Korean placeholders

**Static-heavy pages** (more `data-i18n` attributes needed):
- `episode_upload_pc.html` — form labels, options, buttons
- `novel_upload_pc.html` — form labels, options, buttons
- `auth_pc.html` — login/signup form
- `creator_dashboard_pc.html` — dashboard labels
- `author_dashboard_pc.html` — author info labels

**JS-rendered pages** (mostly `t()` in JS, minimal HTML changes):
- `homepage_pc.html`, `homepage.html` — only nav text + lang picker
- `search_pc.html`, `search.html` — only nav text + lang picker
- `novel_detail_pc.html`, `novel_detail.html` — only nav text + lang picker
- `novel_viewer_pc.html`, `novel_viewer.html` — nav text + reader chrome labels
- `my_library_pc.html`, `my_library.html` — nav text + lang picker
- `payment_pc.html` — nav text + payment labels

## 6. Locale-Aware Formatting

### 6.1 Numbers
Replace all `Intl.NumberFormat("ko-KR")` with `Intl.NumberFormat(inkroadI18n.locale)`.

### 6.2 Currency
- Korean: `"300원"` → `t("common.per_episode") + " " + formatted + t("common.won")`
- English: `"₩300 per ep."` — same keys, different values in en.json

### 6.3 Date/Time
- `formatTime()` in episode-upload.js uses hardcoded "오전"/"오후"
- Replace with `Intl.DateTimeFormat(inkroadI18n.locale, { hour: "numeric", minute: "2-digit" })`

## 7. Edge Cases

- **Flash of untranslated content (FOUC):** `i18n.js` loads and applies synchronously before other scripts. Korean text in HTML serves as default fallback, so worst case is Korean flashing briefly for English users.
- **Missing translation key:** `t()` returns the key string itself and logs console warning. Page remains functional.
- **New language addition:** Add `ja.json` file, add `"ja"` to supported array in i18n.js. No other changes needed.
- **RTL languages:** Not in scope. All current targets (ko, en, ja, zh) are LTR.
- **SEO:** Single-page app with no SSR, so language meta tag (`<html lang="ko">`) should update dynamically: `document.documentElement.lang = locale`.
