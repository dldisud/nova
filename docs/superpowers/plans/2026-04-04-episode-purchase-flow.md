# 2026-04-04 잉크로드 유료 회차 구매 흐름

## 목표

- 작품 상세와 뷰어에서 유료 회차를 실제로 구매할 수 있게 만든다.
- 구매 후에는 같은 계정으로 다시 열었을 때 유료 본문을 바로 볼 수 있게 만든다.
- 구매 내역이 서재에도 반영되게 만든다.

## 범위

### 수정 파일

- `/mnt/d/nova/supabase/schema.sql`
  - 유료 회차 구매용 RPC 추가
- `/mnt/d/nova/assets/store-redesign.js`
  - 상세/뷰어 구매 버튼에 회차 정보와 가격 정보 연결
- `/mnt/d/nova/assets/supabase-live.js`
  - 구매 버튼 처리
  - 로그인 후 복귀 처리
  - 구매 내역 조회 및 서재 반영
- `/mnt/d/nova/my_library_pc.html`
  - 현재 서재 탭 구조와 구매 탭 표시 확인
- `/mnt/d/nova/novel_detail_pc.html`
  - 상세 구매 CTA 연결 확인
- `/mnt/d/nova/novel_viewer_pc.html`
  - 페이월 구매 CTA 연결 확인

### 검증 파일

- `/mnt/d/nova/tests/test_episode_purchase_flow.py`
  - 구매 함수, 구매 버튼 훅, 프론트 호출 연결 여부 확인

## 구현 순서

1. 구매 흐름을 표현하는 최소 회귀 테스트를 먼저 만든다.
2. `purchase_episode` RPC를 추가한다.
3. 상세/뷰어 버튼이 실제 회차 ID를 들고 구매를 요청하게 만든다.
4. 로그인 세션 액세스 토큰을 미러링해서 구매 후 유료 본문 조회가 바로 되게 만든다.
5. 서재 구매 탭을 회차 구매 기준으로 바꾼다.
6. 테스트를 다시 돌려 통과를 확인한다.

## 메모

- 현재 결제는 실결제가 아니라 구매 기록 저장 단계까지만 구현한다.
- 가격은 기존 UI와 맞춰 `편당 고정가 300원 + 세일율 반영`으로 계산한다.
- 구매 후 상세에서는 첫 유료 회차로, 뷰어에서는 현재 회차로 바로 이어진다.
